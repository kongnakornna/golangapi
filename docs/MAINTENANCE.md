# แนวทางการบำรุงรักษา

## การจัดการการกำหนดค่า

- ใช้ `configs/config.yaml` ในเครื่อง

- ใช้ `configs/config.production.yaml` หรือตัวแปรสภาพแวดล้อมในสภาพแวดล้อมการผลิต

- สามารถปิดใช้งาน Redis ได้ผ่าน `app.redis.enabled`

## การย้ายฐานข้อมูล

- เพิ่มไฟล์การย้ายข้อมูล: `migrations/app/*.sql`

- แนะนำให้ย้ายข้อมูลแบบซิงโครนัสหลังจากแก้ไขโมเดลหรือที่เก็บข้อมูล

## การอัปเกรดส่วนประกอบ

- ใช้ `go get -u` เพื่ออัปเกรดส่วนประกอบ

- เรียกใช้ `go test ./...` หลังจากอัปเกรด

- ให้ความสนใจเป็นพิเศษกับการเปลี่ยนแปลงไดรเวอร์ฐานข้อมูลและไลบรารีแคช

## การบันทึกและการตรวจสอบ

- ใช้ `pkg/logger` สำหรับการบันทึกแบบรวมศูนย์

- การตรวจสอบสถานะ: `/health`, `/health/detailed`, `/ready`, `/live`

## ความปลอดภัยและการปฏิบัติตามข้อกำหนด

- คีย์ JWT ต้องระบุผ่านตัวแปรสภาพแวดล้อม

- ตรวจสอบว่านโยบาย CORS และส่วนหัวด้านความปลอดภัยตรงตามข้อกำหนดการใช้งานจริง

- อัปเดตส่วนประกอบต่างๆ อย่างสม่ำเสมอเพื่อแก้ไขช่องโหว่

## การกำหนดเวอร์ชันและการเผยแพร่

- การส่งผลงานเป็นไปตามรูปแบบ `feat:`/`fix:`

- การยืนยันก่อนการเผยแพร่: การทดสอบผ่านแล้ว การกำหนดค่าได้รับการอัปเดตแล้ว Swagger ได้รับการซิงโครไนซ์แล้ว
