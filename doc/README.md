# สารบัญเอกสาร
## ลำดับการอ่านที่แนะนำ
1. `docs/ARCHITECTURE.md` - การออกแบบสถาปัตยกรรมและโครงสร้างไดเร็กทอรี
2. `docs/DEVELOPMENT.md` - กระบวนการพัฒนาและแนวทางการขยาย
3. `docs/MAINTENANCE.md` - การบำรุงรักษาและการจัดการการเปลี่ยนแปลง
4. `docs/DEPLOYMENT.md` - การติดตั้งและการเผยแพร่
5. `docs/MULTI_APP_TEMPLATE.md` - เทมเพลตแอปพลิเคชันหลายตัว
6. `docs/TROUBLESHOOTING.md` - ปัญหาทั่วไปและการแก้ไขปัญหา

## รายชื่อเอกสาร

- `docs/ARCHITECTURE.md`: สถาปัตยกรรมโดยรวม ความรับผิดชอบแบบแบ่งชั้น ห่วงโซ่คำขอ ขอบเขตของโมดูล
- `docs/DEVELOPMENT.md`: การเตรียมสภาพแวดล้อม มาตรฐานการพัฒนา การเพิ่มคุณสมบัติและโมดูล
- `docs/MAINTENANCE.md`: การกำหนดค่าและการย้ายข้อมูล การอัปเกรดส่วนประกอบ การใช้งานที่สำคัญ
- `docs/DEPLOYMENT.md`: กระบวนการและข้อควรระวังในการปรับใช้แบบโลคอล/คอนเทนเนอร์/คลัสเตอร์
- `docs/MULTI_APP_TEMPLATE.md`: เทมเพลตไดเร็กทอรีและโค้ดสำหรับหลายแอปพลิเคชัน
- `docs/TROUBLESHOOTING.md`: การแก้ไขปัญหาทั่วไปและการวินิจฉัยอย่างรวดเร็ว

## คำแนะนำในการบำรุงรักษาเอกสาร

- รักษาความสอดคล้องกันระหว่างเอกสารและโค้ด โดยเฉพาะโครงสร้างไดเร็กทอรี รายการการกำหนดค่า และคำสั่งการทำงาน
- หลังจากเปลี่ยนแปลงเส้นทางหรือโครงสร้างการตอบสนอง โปรดอัปเดตเอกสารและเรียกใช้ `./scripts/swagger.sh`
